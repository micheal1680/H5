代码的模块化可以让代码在不同的场景中重复使用；

  1. 模块化的中心思想是代码的可重用性；
  2. 一般模块是一个独立的JavaScript文件；里面可以是一个或一组相关的类、常用函数等
  3. 模块是为了大规模JS开发;模块中尽可能少定义全局变量,最好不超过一个；

用作命名空间的对象

  1. 在模块创建过程中，用一个对象作为命名空间，可以避免全局变量被污染；
    函数和变量可以作为这个命名空间的属性存储起来， 而不是直接定义全局变量；
  2. 看个命名空间的小例子： 
                        //实例化一个对象
                    var nr = new set.Range(2, 20);
                    console.log(nr);
                    console.log(nr.includes(3));

                    //这里可以自己扩展，当然完全的一般的还是在模块就是range.js文件里扩展
                    set.Range.prototype.sum = function () {
                        var sum = 0;
                        for (var i = this.from; i <= this.to; i++) {
                            sum += i;
                        }
                        return sum;
                    }

                    var m = nr.sum();
                    console.log(m);
